Контекст HRDS для AI-агентов

HRDS AI Kit — это набор файлов, который даёт AI-агенту контекст дизайн-системы HRDS: где брать компоненты, паттерны и токены и по каким правилам работать. Если подтверждённых данных нет, агент пишет Needs review, а не придумывает props.

Ниже — как это устроено и как этим пользоваться. Kit не заменяет Figma, Storybook или код; он направляет агента к проверяемым источникам.

Принцип

Без догадок

Агент обязан проверить source, индекс и rules. Неподтверждённое помечается Unknown или Needs review — не выдаётся за факт.

Source first

Canonical links в knowledge/sources.md. Figma node URL для design-to-code.

Rules before output

AI, Figma, Storybook, components, patterns, tokens, accessibility — task-specific guardrails.

Gaps, not guesses

Если props или story не подтверждены, агент перечисляет gaps вместо выдуманного API.

Поведение агента

Что меняется в ответах

Разница между агентом без контекста и агентом, который читает kit перед ответом.

Без контекста
  • придумывает props и variants
  • рисует generic layout вместо HRDS
  • расходится со Storybook
  • отвечает без проверки source
→
С контекстом HRDS
  • берёт props только из Storybook/кода
  • использует компоненты библиотеки HRDS
  • сверяет Figma ↔ Storybook ↔ код
  • перечисляет gaps вместо догадок

Как использовать

Три шага, чтобы начать

Kit — это папка с файлами. Её кладут рядом с проектом, чтобы агент видел и код, и контекст HRDS.

01

Скачать и распаковать

Положите папку hrds-ai-kit рядом с продуктовым репозиторием.

02

Добавить в workspace

File → Add Folder to Workspace. Теперь агент видит и продукт, и контекст HRDS в одном окне.

03

Вызвать skill

В чате наберите /component-checker, /pattern-checker или /technical-analyst. Для Figma передайте node-specific URL.

Ниже — пример запроса. Скопируйте, подставьте свою ссылку на Figma-node и отправьте агенту.

Проверь Figma frame по HRDS: components, patterns, tokens, accessibility. Node: https://www.figma.com/design/...?node-id=123-456 Если props не подтверждены — перечисли gaps, не выдумывай API.
FAQ

Частые вопросы

Это замена Figma или Storybook?

Нет. Kit — это context layer для AI. Он не хранит дизайн-систему, а указывает агенту на источники и правила работы с ней.

Что значит статус Starter?

Это неподтвержденные данные: их нужно проверить по Figma, Storybook и коду. Агент не выдаёт их за source of truth.

С какими инструментами работает?

Cursor, Claude Code, Copilot, Codex и любые MCP-агенты. Достаточно положить папку в workspace.

Можно ли генерировать production-код?

Только с human review. Design-to-code и изменения API требуют подтверждения по source.

Limits

Ограничения

Не production-каталог

Многие entries всё ещё Starter. Props и Storybook stories нужно подтверждать.

Не автопилот

Design-to-code требует human review, если нет прямого source.

С чего начать

Скачайте архив, положите папку рядом с репозиторием и добавьте в workspace. Дальше вызывайте skills через / и передавайте агенту node-specific ссылки на Figma. Всё остальное описано выше.
