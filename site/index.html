Каждый frame без контекста стоит часов. Каждый раз.

Лёгкое вроде бы: «опиши экран по HRDS». А дальше — агент выдумывает variants, путает паттерны, не видит detached-инстансы и не сверяет props со Storybook. Разработчик получает spec с догадками. Дизайнер — review с теми же замечаниями снова.

Это та работа, на которую никто не закладывает время — и которую все делают заново на каждом экране. Часы уходят до того, как начинается реальная задача.

HRDS AI Kit — семантический слой над дизайн-системой HR Tech. Сотни выборов — источники, mapping, статусы, skills — зафиксированы в v0.2 pilot. Не каталог на замену Storybook. Положи рядом с репо, добавь в workspace — маршрут готов.

  1. 001 / Source-first routing

    Canonical links в knowledge/sources.md. Figma node URL обязателен для design-to-code. Storybook 13.3.0 и @yandex-int/hr-components — для API.

  2. 002 / Component mapping

    Явная цепочка: Figma property → Storybook control → code import → example. Без источника — Needs review, не догадка.

  3. 003 / Agent-native

    Попроси агента описать HRDS с нуля — получишь другой API каждый раз. Здесь решения приняты: rules/ + skills/ + .cursor/skills/. Без drift.

  4. 004 / Version drift detection

    component-checker ловит detached, deprecated и foreign instances. Сверка с библиотекой 13.3.0 и decision-log.

  5. 005 / Pattern fit

    pattern-checker отделяет Page Header от Content Header, Empty State от Error — и подсказывает замену.

  6. 006 / Frame spec для разработки

    technical-analyst выдаёт handoff: components, props, states, tokens, behavior, gaps — без выдуманной логики.

  7. 007 / Gaps, not guesses

    Starter и Proposed явно помечены. Confirmed — только имя из Figma metadata. Props не выдаются за факт без Storybook/code.

  8. 008 / Подключение за три шага

    Скачать → Add Folder to Workspace → вызвать /component-checker или /design-review с node-specific URL.

002 / Метки в выводе

Confirmed

Есть источник: Figma metadata, Storybook story или code. Опирайся, но сверь перед prod.

Starter

Данные из kit, не проверены. Агент не выдаёт за факт — помечает и просит верификацию.

Needs review

Prop без подтверждённого источника. Найди story или код — не додумывай.

Proposed

Гипотеза по формату examples. Явно не факт. Рядом — «сверить в Storybook».

003 / Приоритет источников

При расхождении — в таком порядке. Агент фиксирует конфликт, не разрешает молча.

  1. 01Code / Storybook — props, controls, behavior
  2. 02Figma component — структура, variants, properties
  3. 03Rules — accessibility, naming, review
  4. 04Examples — формат вывода, не source of truth
  5. 05Decision log — принятые исключения

004 / До и после · Button

Один компонент. Слева — типичный ответ без kit. Справа — с метками и gaps.

без kit

Button на экране:
- variant: primary | secondary | tertiary | outline
- size: sm | md | lg | xl
- loading: boolean

Props: Type=Primary, Size=Medium
→ <Button variant="primary" size="md" />

с kit

Button · Confirmed (имя из Figma)

variant: primary | secondary | ghost | danger
  [Proposed — Needs review]
size: md  [Proposed]
states: default, hover, focus, disabled, loading

Gaps: API loading — Needs review
Open: story в Storybook 13.3.0

Перед подключением

Q.001 / Это замена Figma или Storybook?

Нет. Kit — context layer для AI. Он указывает агенту на источники и правила. Дизайн и API остаются в Figma и Storybook.

Q.002 / Чем отличается от «просто скинуть ссылку на Storybook»?

Storybook — source of truth для API, но не маршрут для агента. Kit фиксирует приоритеты, статусы Starter/Confirmed, skills для review и handoff, и запрещает выдавать гипотезы за факты.

Q.003 / Работает ли с Cursor?

Да. Skills подключены через .cursor/skills/. Вызов: /component-checker, /pattern-checker, /technical-analyst, /design-review, /prototype-builder.

Q.004 / Можно ли генерировать production-код?

Только с human review. Design-to-code требует подтверждения props по Storybook или коду. Kit снижает догадки, не снимает ответственность.

Q.005 / Что значит Starter?

Неподтверждённые данные. Агент не выдаёт их за source of truth. После проверки по Figma/Storybook/code статус обновляется на Confirmed.

Q.006 / Какая версия библиотеки?

Актуальный Storybook: 13.3.0. Пакет: @yandex-int/hr-components. Version drift проверяет component-checker.

Q.007 / Что kit не делает?

Не полный каталог HRDS. Не автопилот. Не замена human DS review. Не доказательство существования prop без источника.
